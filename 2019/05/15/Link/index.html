Linux link 相關筆記

軟連結、硬連結

inode

inode這東西可以想像成指標（pointer），在Linux系統中，他會指向硬碟的某個物理位置

硬連結（Hard link）:

同一個inode值，如果有其他link也是這個值，就是這個inode的 硬連結，硬連結數量 可以多個，刪除其一會減少連結數量。只有連結數量為1時刪除會刪除整個檔案。

軟連結（Symbolic link）:

指向一個絕對路徑或相對路徑，刪除軟連結本身，不會影響目標檔案。若目標檔案被刪除或被移動，軟連結本身不會變動，這軟連結可以稱為 被遺棄。

從上面可以看到，軟連結只會顯示 ->，硬連結不會特別顯示這個箭頭，但硬連結的inode值會與指向的目標inode值相同。

ln — 建立連結指令

預設都是建立硬連結（Hard link）

ln 目標的路徑 連結名稱

若要建立軟連結（Symbolic link）: ln -s 目標的路徑 連結名稱

unlink 刪除連結

unlink 連結名稱

ln 常用參數
